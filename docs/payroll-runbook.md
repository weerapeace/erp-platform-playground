# Payroll Runbook

คู่มือสั้นสำหรับทำเงินเดือน 1 รอบใน ERP Playground

## ลำดับการทำงาน

1. ตั้งค่างวดเงินเดือน
   - สถานะเริ่มต้นควรเป็น `draft`
   - ต้องมีวันที่เริ่ม, วันที่สิ้นสุด, วันทำงาน, และชั่วโมงต่อวัน

2. ตรวจข้อมูลพื้นฐาน
   - พนักงานต้องเป็น active
   - สัญญาปัจจุบันต้องเป็น active/current
   - เงินเดือนหรือค่าแรงต้องมากกว่า 0
   - รายการเงินประจำต้องผูกพนักงาน และมีวันที่เริ่ม

3. กรอกข้อมูลประจำงวด
   - สาย / ออกก่อน
   - ขาด
   - ลา
   - OT
   - งานเหมา
   - เพิ่มพิเศษ / หักอื่น

4. ไปหน้า "คำนวณงวด"
   - ดูกล่อง "ตรวจความพร้อมงวด"
   - ถ้ามี error ต้องแก้ก่อน ระบบจะไม่ให้คำนวณหรือบันทึก
   - ถ้ามี warning ยังทำต่อได้ แต่ควรตรวจดู

5. กด "คำนวณ + เทียบ"
   - ระบบจะคำนวณแบบ background job
   - เมื่อผลออกแล้วให้ดูยอดต่างจากรอบก่อนหน้า

6. กด "บันทึกผลคำนวณ"
   - ระบบจะสร้าง payroll run ใหม่
   - ไม่ลบ run เก่า เพื่อให้ตรวจย้อนหลังได้

7. ไปหน้า "ตรวจสอบเงินเดือน"
   - ตรวจยอดรวมทั้งงวด
   - ใช้ตัวกรองเคสเสี่ยง เช่น สุทธิติดลบ, หักเกิน 50%, ไม่มีฐานเงิน, วันทำงาน 0
   - ถ้าต้องแก้ ให้ส่งกลับไปแก้/คำนวณใหม่

8. อนุมัติและล็อกงวด
   - กด "อนุมัติผลเงินเดือน" เมื่อยอดถูกต้อง
   - กด "ล็อกงวด" เพื่อกันแก้ตัวเลขโดยไม่ตั้งใจ

9. ออกสลิป
   - กด "ออกสลิป" จากหน้า Review หรือไปหน้า "สลิปเงินเดือน"
   - ถ้ามีสลิปเดิม ระบบจะอัปเดต ไม่สร้างซ้ำ

10. ส่งไปจ่าย / จ่ายแล้ว
    - ใช้หลังล็อกงวดและออกสลิปแล้ว
    - API จะกันไม่ให้ mark paid ถ้ายังไม่มีสลิป

## Validation แปลว่าอะไร

- งวดไม่พร้อมคำนวณ: สถานะงวดไม่ใช่ draft/review หรือข้อมูลวันที่ไม่ครบ
- วันทำงานว่าง: งวดยังไม่ได้ตั้งจำนวนวันทำงาน
- ชั่วโมงต่อวันว่าง: งวดยังไม่ได้ตั้งชั่วโมงต่อวัน
- ไม่พบสัญญาที่ใช้คำนวณ: ไม่มีสัญญา active/current ตรงกับบริษัทของงวด
- ค่าแรงไม่พร้อม: สัญญามีเงินเดือน/ค่าแรงเป็น 0 หรือว่าง
- รายการประจำไม่ผูกพนักงาน: เงินเพิ่ม/หักประจำยังไม่ได้เลือกพนักงาน
- รายการประจำยอดไม่ถูกต้อง: ยอด/จำนวน/อัตราเป็น 0
- เคยคำนวณแล้ว: เป็นคำเตือนเท่านั้น ระบบจะสร้าง run ใหม่ ไม่ลบของเดิม

## กติกาที่ระบบบังคับ

- คำนวณและบันทึกได้เฉพาะงวด `draft` หรือ `review`
- อนุมัติ/ล็อก/จ่าย ต้องมีผลคำนวณและบรรทัดเงินเดือนแล้ว
- ทำเครื่องหมายจ่ายแล้วต้องมีสลิปก่อน
- ทุกการเปลี่ยนสถานะผ่าน API กลางและมี audit log
- เงินประจำลบถาวรไม่ได้ ให้ปิดใช้งานแทนเพื่อเก็บประวัติ
